weave

A small, verifiable language for orchestrating AI agents.

A whole program

This is real Weave. It drafts with a live model and refuses to return anything that breaks the rules. Orchestration and verification, in one file.

blue arrows bind a step's result to a name (orchestration) require / ensure are contracts, checked and self-corrected (verification)

Five concepts, the whole language

type
A shape for data: records, unions, lists. Where contracts attach.
tool
A typed door to the outside: an MCP tool, a CLI, an HTTP call, a function.
agent
A configured model caller: which model, persona, tools, retry budget.
flow
The program. Typed inputs and outputs, a body of steps, contracts.
soft call
Calling an agent with a prompt and getting a typed result back.

The mechanic that makes it worth it

When a contract fails on a model's output, Weave does not just error. It feeds the violated contract back to the agent and retries. Guardrail and self-correction, in two lines.

In a live run, the model's first draft was over 280 characters, the contract caught it, and the retry came back under the limit. The language drove the fix.

Why a language for AI even makes sense

Not because a model writes it more fluently (a new language has no training data, so it cannot). Because it is small enough to teach in a prompt, constrained enough that broken programs cannot be expressed, and strict enough that wrong output is caught the instant it happens. Trust, not fluency.
